# 题炼产品重构实施计划

- [x] 1. 重写产品与工程文档
  - 将 PRD 的定位、用户、核心流程、范围和验收标准调整为学生基础刷题与文档生成题库
  - 将 SDD 的架构、数据模型、接口、权限和部署说明同步到已确认设计
  - 更新项目 README、产品名称、功能列表、环境变量和运行说明
  - _需求：R1、R2、R3、R4、R5、R6_

- [x] 2. 建立数据库迁移和领域模型
  - 为用户增加 `student|admin` 角色
  - 为题库增加平台/文档来源及草稿/就绪状态
  - 将题型约束收敛到单选和多选，并去除新增题目对知识点的依赖
  - 新增源文档与生成任务模型
  - 为题目统计、用户资源查询和生成任务建立索引
  - 创建兼容现有 SQLite 数据的 Alembic 迁移
  - _需求：R1、R3、R4、R5、R6_

- [x] 3. 修复认证与资源权限边界
  - 移除题库和题目接口中写死的用户 ID
  - 为学生、管理员和资源所有者建立统一依赖校验
  - 限制学生只能维护自己的个人题库，平台题库仅管理员可维护
  - 将 CORS 来源改为环境变量配置
  - _需求：R1、R3、R5、R6_

- [x] 4. 完善基础题库和题目接口
  - 支持按平台题库与个人题库分区查询
  - 实现个人题库及其题目的查看、修改和删除
  - 过滤旧简答题，保证新刷题流程仅出现单选与多选
  - 为题库详情返回题量、题型和可操作权限
  - _需求：R1、R5_

- [x] 5. 实现管理员平台题库导入
  - 定义并提供 UTF-8 JSON 导入模板
  - 实现全量结构与业务规则校验接口
  - 实现单事务导入与错误整体回滚
  - 为管理员导入权限和错误定位编写测试
  - _需求：R6_

- [x] 6. 实现文档上传与解析
  - 接入 PDF、DOCX、TXT、Markdown 解析依赖
  - 实现 10 MB、扩展名、随机存储名、用户隔离和空文本校验
  - 区分不支持格式、文件过大、无可提取文本和解析失败错误
  - 为四种文档和异常输入编写测试
  - _需求：R3_

- [x] 7. 实现文档题库生成任务
  - 将 AI Provider 改为依据文档生成单选/多选题，移除解释与简答题路径
  - 强制单次生成 1～20 题，并校验题干、四个选项、答案和解析
  - 实现任务创建、状态查询、失败重试和草稿保存
  - 未配置模型时返回明确错误，不生成占位题
  - 确保日志不包含文档正文、令牌或 AI 密钥
  - _需求：R3、R4_

- [x] 8. 实现生成题预览、编辑与确认
  - 实现草稿题修改和删除接口
  - 实现任务状态转换和重复确认保护
  - 使用数据库事务将确认结果创建为个人题库
  - 为越权、非法状态和失败回滚编写测试
  - _需求：R4、R5_

- [x] 9. 完善刷题判分与正确率统计
  - 校验题目属于当前会话并防止同题重复提交
  - 单选使用唯一答案判分，多选使用答案集合完全一致判分
  - 提交后返回包含本次作答的个人历史与全网正确率、正确次数及样本数
  - 为首次作答、多次练习、多用户统计和重复请求编写测试
  - _需求：R2_

- [x] 10. 重构学生端信息架构与品牌
  - 全站正式更名为“题炼”
  - 移除知识点解释和输入知识点出题导航，将旧路由重定向至文档生成页
  - 实现平台题库与我的题库分区首页及题库维护入口
  - 按确认的校园讲义式设计更新颜色、字体、布局和响应式导航
  - _需求：R1、R5_

- [x] 11. 实现学生文档生成界面
  - 实现文件上传、题库名称、题型数量与难度配置
  - 实现生成状态轮询、失败信息和重试
  - 实现题目预览、编辑、删除、确认和立即刷题
  - 对 AI 产生的 Markdown 内容进行安全净化
  - _需求：R3、R4_

- [x] 12. 完善刷题界面与统计反馈
  - 保留顺序和随机练习模式
  - 提交后展示判题、标准答案、解析、个人历史正确率和全网正确率
  - 完成后展示本次总题数、正确题数和正确率
  - _需求：R2_

- [x] 13. 实现管理员导入界面
  - 仅对管理员展示入口
  - 实现 JSON 文件选择、校验结果、错误定位和确认导入
  - 普通学生访问管理路由时跳转到题库首页
  - _需求：R6_

- [x] 14. 执行全量验证与收尾
  - 运行后端测试并修复失败
  - 运行前端构建和静态检查
  - 启动前后端并在浏览器验证基础题库刷题、文档生成题库、正确率展示及管理员导入
  - 检查前端控制台、权限越界、上传限制和相邻路由回归
  - 更新任务状态并记录无法验证的真实 AI 外部依赖
  - 浏览器实例在当前环境不可用；已完成 API 集成测试与生产构建，真实 AI 成功路径需提供模型配置后验证
  - _需求：R1、R2、R3、R4、R5、R6_
