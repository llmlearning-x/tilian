# 题炼错题本功能升级设计方案

> 设计目标：在现有「自动收集错题 + 斩题」基础上，构建一套可管理、可复习、可分析的错题本体系。
> 设计原则：不破坏现有数据模型，优先复用 `UserQuestionStat`，通过少量扩展字段与独立视图满足管理需求。

---

## 一、现状回顾

### 1.1 数据模型
`UserQuestionStat` 已包含以下关键字段：

| 字段 | 含义 |
|------|------|
| `user_id` / `question_id` / `bank_id` | 用户-题目-题库关联 |
| `correct_count` / `wrong_count` | 对/错次数 |
| `is_mastered` | 是否斩题（已掌握） |
| `last_result` | 最近作答结果 |
| `last_answer_at` | 最近作答时间 |
| `mastered_at` | 斩题时间 |

### 1.2 已有接口
- `GET /api/quiz/wrong-questions?bank_id=`：错题列表
- `POST /api/quiz/master`：斩题 / 取消斩题
- `GET /api/quiz/mastered-questions?bank_id=`：已掌握列表

### 1.3 当前页面能力
- 平铺展示所有错题
- 支持按题库筛选（URL 参数）
- 单题「查看解析」「斩题」

---

## 二、功能升级方案

### 2.1 按题库组织错题（核心诉求）

**方案：双视图切换**
1. **题库维度视图**：先展示「有错题的题库」卡片，点击后进入该题库的错题列表。
2. **题目维度视图**：保留现有平铺列表，增加更丰富的筛选与排序。

**后端支持**
- 新增接口 `GET /api/quiz/wrong-questions/banks`：返回当前用户有错题的题库聚合信息。
  ```json
  [
    {
      "bank_id": 4,
      "bank_name": "阿里云ACP认证考试题库",
      "wrong_count": 12,
      "mastered_count": 3,
      "last_wrong_at": "2026-07-08T06:14:12",
      "cover_question_stem": "企业中，为了提高数据的安全性..."
    }
  ]
  ```

**前端交互**
- 默认进入「题库维度」：卡片展示题库名称、错题数、已斩题数、最近错误时间。
- 点击卡片进入该题库的错题列表。
- 提供「全部错题」入口，平铺跨题库所有错题。

---

### 2.2 错题筛选与排序

**筛选条件**
- 按题库（单选 / 多选）
- 按题型：单选题 / 多选题
- 按错误次数：1 次 / 2 次 / 3 次及以上
- 按最近结果：答错 / 答对（仍保留在错题本）
- 按斩题状态：未掌握 / 已掌握
- 按关键词搜索题干

**排序方式**
- 最近答错时间（默认）
- 错误次数从高到低
- 题库名称
- 正确率相关（可计算 `wrong_count / (correct_count + wrong_count)`）

**后端支持**
- 扩展 `GET /api/quiz/wrong-questions`：
  - Query 参数：`bank_id`, `type`, `min_wrong_count`, `last_result`, `is_mastered`, `keyword`, `sort_by`, `order`
  - 分页参数：`page`, `page_size`

---

### 2.3 错题练习模式

**需求**：从错题本中抽题进行专项练习，而非只能查看解析。

**方案**
- 新增「错题重练」入口：
  - 全部错题重练
  - 当前题库错题重练
  - 筛选后的错题重练
- 后端新增接口 `POST /api/quiz/practice/wrong-questions`：
  - 接收 `bank_id`, `question_ids`, `mode`（顺序/随机）
  - 为错题创建一次新的 `QuizSession`，复用现有练习流程
  - 返回 `session_id`，前端跳转 `/practice/session/:session_id`

**价值**：错题本不再是静态查看，而是真正用于巩固。

---

### 2.4 批量操作

**功能**
- 批量斩题：选中多题后一键标记为已掌握。
- 批量移出错题本：将选中题目的 `wrong_count` 清零（保留 `correct_count` 统计，但不再显示）。
- 批量加入错题重练。

**后端支持**
- `POST /api/quiz/wrong-questions/batch-master`：批量斩题
- `POST /api/quiz/wrong-questions/batch-clear`：批量移出错题本

**前端交互**
- 列表项左侧显示复选框。
- 顶部/底部显示批量操作栏。

---

### 2.5 错题统计与薄弱点分析

**功能**
- 错题总数、已掌握数、待复习数。
- 按题库的错误分布（饼图 / 柱状图）。
- 高频错题 Top N。
- 最近 7 天 / 30 天错题趋势。

**后端支持**
- 新增接口 `GET /api/quiz/wrong-questions/statistics`：
  ```json
  {
    "total_wrong": 15,
    "mastered": 3,
    "pending": 12,
    "by_bank": [
      { "bank_id": 4, "bank_name": "阿里云ACP...", "wrong_count": 12, "mastered_count": 2 }
    ],
    "top_questions": [
      { "question_id": 1, "stem": "...", "wrong_count": 5 }
    ],
    "trend": [
      { "date": "2026-07-01", "wrong_count": 2 }
    ]
  }
  ```

**前端交互**
- 在错题本首页顶部展示统计卡片。
- 可切换至「分析」标签页查看图表。

---

### 2.6 错题详情与笔记

**功能增强**
- 错题详情弹窗：完整题干、选项、正确答案、解析、用户历次作答记录。
- 用户可添加「错题笔记」，记录错误原因、知识点总结。

**数据模型扩展（可选）**
- 新增 `UserWrongQuestionNote` 表：
  - `id`, `user_id`, `question_id`, `note`, `created_at`, `updated_at`
- 或在 `UserQuestionStat` 中新增 `note` 字段（简单场景）。

**后端支持**
- `GET /api/quiz/wrong-questions/{question_id}`：错题详情（含作答历史）
- `PUT /api/quiz/wrong-questions/{question_id}/note`：保存笔记

---

### 2.7 错题导出

**功能**
- 导出当前筛选结果或全部错题为：
  - Markdown / Word 文档（便于打印复习）
  - JSON（便于备份或迁移）

**后端支持**
- `POST /api/quiz/wrong-questions/export`：
  - 接收筛选参数与 `format`
  - 返回文件下载流

---

### 2.8 复习计划（高阶）

**功能**
- 基于艾宾浩斯遗忘曲线，为每道错题推荐下次复习时间。
- 每日生成「今日待复习」列表。

**数据模型扩展**
- 在 `UserQuestionStat` 新增 `next_review_at` 字段。
- 斩题后清零，答错后根据错误次数计算下次复习时间。

**后端支持**
- `GET /api/quiz/wrong-questions/today-review`：今日待复习列表
- 在提交答案时更新 `next_review_at`

## 三、页面结构建议

```
/wrong-questions
├── 统计卡片区（总数 / 待复习 / 已掌握 / 今日待复习）
├── 视图切换：题库视图 | 全部错题 | 分析
├──
├── 题库视图
│   └── 题库卡片网格（名称 / 错题数 / 已掌握 / 最近错误）
│       └── 点击进入 /wrong-questions?bank_id=4
│
├── 全部错题视图
│   ├── 筛选栏（题库 / 题型 / 错误次数 / 状态 / 关键词）
│   ├── 批量操作栏（选中 N 项：批量斩题 / 批量移出 / 错题重练）
│   └── 错题卡片列表
│       └── 单题操作：查看解析 / 斩题 / 重练 / 笔记
│
└── 分析视图
    ├── 错题分布（按题库）
    ├── 高频错题 Top 10
    └── 错题趋势
```

## 五、数据库变更（建议）

### 最小化变更（推荐先做）
仅扩展 `UserQuestionStat` 表：
```sql
ALTER TABLE user_question_stats ADD COLUMN note TEXT;
ALTER TABLE user_question_stats ADD COLUMN next_review_at DATETIME;
```

### 独立笔记表（如果笔记功能复杂）
```sql
CREATE TABLE user_wrong_question_notes (
    id INTEGER PRIMARY KEY,
    user_id INTEGER NOT NULL,
    question_id INTEGER NOT NULL,
    note TEXT NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(user_id, question_id)
);
```

## 六、实施阶段建议

### 第一阶段：基础管理能力（推荐优先做）
1. 按题库聚合接口与题库视图。
2. 错题列表增加筛选、排序、分页。
3. 批量斩题 / 批量移出错题本。

### 第二阶段：复习闭环
1. 错题重练功能。
2. 错题详情弹窗 + 作答历史。

### 第三阶段：分析与高阶
1. 错题统计仪表盘。
2. 错题笔记。
3. 复习计划（next_review_at）。
4. 导出功能。

---

## 七、风险与注意事项

1. **性能**：错题列表涉及 `UserQuestionStat` 与 `Question`、`QuestionBank` 多表关联，增加筛选和分页后建议为 `user_id + wrong_count + is_mastered` 建立复合索引。
2. **数据一致性**：批量操作和错题重练应放在同一事务中。
3. **权限**：所有接口必须校验 `user_id`，防止越权访问他人错题。
4. **兼容**：扩展 `GET /api/quiz/wrong-questions` 时保持现有默认行为不变。
